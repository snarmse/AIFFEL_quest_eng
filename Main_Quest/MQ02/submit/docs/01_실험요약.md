# 풍력 터빈 블레이드 결함 탐지 — 실험 요약

## 1. 프로젝트 개요

드론 이미지에서 **Dirt(오염)** / **Damage(손상)** 탐지 → 심각도 등급 판정 → 관리자 에스컬레이션 알림

- **데이터셋**: NordTank 586×371 (Kaggle)
- **모델**: YOLOv8s (최종 선정)
- **핵심 지표**: Damage Recall (FN = 사고 직결)

---

## 2. 데이터셋

| 구분 | 수량 |
|---|---|
| 전체 이미지 | 13,470장 |
| 결함 이미지 (라벨 있음) | 2,995장 |
| 정상 이미지 (라벨 없음) | 10,475장 |
| Dirt 박스 (class 0) | 581개 |
| Damage 박스 (class 1) | 8,770개 |

- Damage : Dirt 비율 = **15 : 1** (심각한 클래스 불균형, 8,770 : 581)
- Damage 객체의 **86%가 64×64px 이하** 소형 객체 (너비·높이 모두 기준, 7,513/8,770)
- 외부 데이터: Wind Turbine Blade v7i (Roboflow) — crack(0) → Damage(1) 리맵, train 1,474장만 사용

### 데이터셋 구성 비교

| 데이터셋 | train | val |
|---|---|---|
| dataset_yolo_full (베이스라인) | 10,776 | 2,694 |
| dataset_yolo_full_aug (+ Dirt×8 증강) | 14,400 | 2,694 |
| dataset_yolo_full_blade (+ blade) | 12,250 | 2,694 |
| dataset_yolo_full_aug_blade (+ Dirt×8 + blade) | 15,874 | 2,694 |

> val 2,694장은 4개 실험 모두 동일 → 공정 비교 가능

## 3. 평가 지표 설계

| 순위 | 지표 | 이유 |
|---|---|---|
| 1 | **Damage Recall** | FN(미탐) = 사고 직결 |
| 2 | **Damage AP50** | 손상 탐지 종합 성능 |
| 3 | mAP50 | 전체 탐지 성능 (참고) |

---

## 4. Phase 1 — 모델 선정 (labeled-only, 20 epochs)

| 모델 | mAP50 | Recall | Precision |
|---|---|---|---|
| YOLOv8n | 0.794 | 0.747 | 0.818 |
| **YOLOv8s ← 선정** | **0.818** | 0.750 | **0.854** |
| YOLO11n | 0.788 | 0.741 | 0.812 |
| YOLO11s | 0.799 | 0.746 | 0.851 |

> **YOLOv8s 선정 이유**: 드론(엣지 디바이스) 탑재 목적으로 정확도와 추론 속도의 균형점. YOLOv8s는 nano 대비 mAP +2.4%p, 모델 크기는 22MB로 실용적.

---

## 5. Phase 5 — 데이터셋 비교 (YOLOv8s, 50 epochs, val 2,694)

| 실험 | 변경 변수 | Precision | Recall | mAP@0.5 | mAP@0.5:0.95 | Damage AP@0.5 |
|---|---|---|---|---|---|---|
| **베이스라인** | full dataset | 0.710 | **0.769** | **0.752** | 0.517 | **0.688** |
| 실험1 | + Dirt×8 증강 | 0.674 | 0.751 | 0.718 | 0.515 | 0.684 |
| 실험2 | + Blade 외부 데이터 | 0.710 | 0.763 | 0.748 | 0.517 | 0.688 |
| 실험3 | + Dirt×8 + Blade | 0.645 | 0.735 | 0.726 | **0.525** | 0.682 |

### 실험 해석

- **실험1 (Dirt×8 증강)**: Dirt 클래스에 치우친 학습 → Damage Recall, Precision 동반 하락
- **실험2 (Blade 추가)**: 도메인 상이(384×384, 다른 터빈) → NordTank 성능에 기여 없음
- **실험3 (증강+Blade)**: 두 변화가 시너지 없이 노이즈로 작용 → 전 지표 최저

## 6. 최종 모델

```
모델 가중치 : runs/exp_full_small_ep50/weights/best.pt
추론 conf   : 0.20
```

### conf=0.2 선정 근거

- Damage의 86%가 소형 객체 → conf를 낮춰 저신뢰도 탐지 포함
- `model.val()` 비교 결과, conf 0.20~0.25 구간에 **실제 TP 존재** 확인 (mAP50: 0.7016 → 0.7137)

### 최종 성능 (val 2,694장, conf=0.001 기준)

| mAP50 | mAP50-95 | Precision | Recall | Damage Recall | Damage AP50 |
|---|---|---|---|---|---|
| 0.752 | 0.517 | 0.710 | 0.769 | 0.645 | 0.688 |

## 7. 서비스 설계 (데모 연동 예정)

### 심각도 스코어링

```
터빈 위험도 = Σ(클래스가중치 × 크기가중치 × confidence)

클래스가중치 : Damage=10, Dirt=2
크기가중치   : 1 + (박스 면적비율 × 5)
```

### 등급 분류

| 점수 | 등급 | 알림 |
|---|---|---|
| 0 | 🟢 정상 | 로그만 기록 |
| 1~15 | 🟡 관찰 | 대시보드 표시 |
| 15~40 | 🟠 주의 | 대시보드 + 이메일 |
| 40+ | 🔴 위험 | Slack 즉시 경보 |

### 오탐 관리 (human-in-the-loop)

| confidence | 처리 |
|---|---|
| ≥ 0.7 | 자동 경보 |
| 0.4 ~ 0.7 | ⚠️ 재확인 필요 플래그 |
| < 0.4 | 무시 |

## 4. 왜 이렇게 결정했는가 (핵심 근거)

### 평가 지표: 왜 Recall 우선인가

FN(미탐) 1건 = 손상을 놓침 = 설비 사고 직결.
FP(오탐)는 재확인 플래그로 운영 레벨에서 보완 가능.
→ 미탐 비용 >> 오탐 비용이므로 Recall 최우선.

mAP50은 Dirt/Damage를 동등 평균하기 때문에, Dirt를 잘 잡아도 값이 올라간다. Damage를 놓쳐도 좋은 모델처럼 보이는 구조라 주 지표로 쓰지 않았다.

### 왜 증강·외부 데이터가 역효과였나

- **Dirt×8 증강**: Dirt 샘플 과다 → Damage 학습 신호 희석 → Recall, Damage AP50 동반 하락
- **blade 외부 데이터**: 해상도 다름(384×384 vs 586×371), 다른 터빈 기종 → 도메인 불일치 → 노이즈로 작용

### 왜 conf=0.20인가

`model.val()`은 내부적으로 threshold를 sweep하기 때문에 conf를 바꿔도 Recall이 변하지 않는다. 대신 mAP50이 0.7016 → 0.7137로 개선된 것으로 해당 구간에 실제 TP가 존재함을 확인. `predict()` 실제 추론에서는 conf=0.20이 적용되어 소형 객체 탐지가 포함된다.

## 5. 서비스 설계

### 심각도 스코어링

```
터빈 위험도 = Σ(클래스가중치 × 크기가중치 × confidence)

클래스가중치: Damage=10, Dirt=2
크기가중치:   1 + (박스 면적비율 × 5)
```

### 등급 및 에스컬레이션

| 점수 | 등급 | 알림 | 대상 |
|---|---|---|---|
| 0 | 🟢 정상 | 로그만 | — |
| 1~15 | 🟡 관찰 | 대시보드 | 현장 담당자 |
| 15~40 | 🟠 주의 | 대시보드 + 이메일 | 담당자 + 팀 |
| 40+ | 🔴 위험 | Slack 즉시 경보 | 담당자 + 관리자 |

### 오탐 관리 (Human-in-the-loop)

| confidence | 처리 |
|---|---|
| ≥ 0.7 | 자동 경보 |
| 0.4 ~ 0.7 | ⚠️ 재확인 필요 플래그 |
| < 0.4 | 무시 |

## 참고 — 취합 시 유의사항

- **평가 기준이 두 가지다**: 모델 선정 실험은 자체 val 사용. 데이터셋 비교(Phase 2)는 full val 2,694장 기준. 발표에서 혼용하지 않도록 주의.
- **최종 모델은 full_small_ep50이다**: 실험 과정에서 full_aug_blade가 중간 최우수로 기록된 시점이 있으나, 최종 full 비교에서 베이스라인(full)이 전 지표 최고로 확인됨.
- **conf=0.20은 inference 전용 파라미터다**: 학습 중에 영향 없음. 추론(predict)에서만 적용.
